:::info Gilt für:
Dieser Artikel richtet sich an Studierende, die kurz vor dem Beginn ihrer Bachelor- oder Masterarbeit stehen.
:::


## Datenproduktion

:::info Wie:           
Während der Arbeit im Labor für eine Bachelor- oder Masterarbeit werden viele Daten produziert, zum Beispiel das Reaktionsprotokoll oder die analytischen Daten oder [Metadaten](/docs/metadata).
:::

:::note Hinweis:        
Wenn Sie mehr über domänenspezifische Datenproduktionsmethoden erfahren möchten, können Sie sich auch die [domänenspezifischen Profile](/docs/domain_guide) ansehen.
:::

## Datendokumentation

:::danger Warum:       
Die Verwendung von [gut organisierten](/docs/data_organisation) und [dokumentierten Daten](/docs/data_documentation) spielt eine wichtige Rolle beim Auffinden, Suchen und Verwenden von Forschungsdaten während des gesamten [Datenlebenszyklus](/docs/data_life_cycle).
:::

:::info Wie:         
Für die [Dokumentation dieser Daten](/docs/data_documentation) kann ein klassisches Laborjournal auf Papier oder ein [elektronisches Laborjournal](/docs/eln) (ELN) verwendet werden.
:::

:::caution Überlegungen:              
Die Verwendung eines papierbasierten Laborjournals scheint eine einfache und effiziente Art der [Datendokumentation](/docs/data_documentation) zu sein. Die Verwendung eines [ELN](/docs/eln) wie [Chemotion](https://www.chemotion.net/chemotionsaurus/index.html) hat jedoch auf lange Sicht viele Vorteile. Zum Beispiel können in [Chemotion](https://www.chemotion.net/chemotionsaurus/index.html) alle aus einem Experiment gewonnenen Daten mit Ihrem Protokoll verknüpft werden, was einen einfachen Weg darstellt, um am Ende der Arbeit einen [gut organisierten Forschungsdatensatz](/docs/data_organisation) zu haben. Zusätzlich kann das Reaktionsprotokoll in einer [standardisierten Form](/docs/format_standards) exportiert werden, die sich für die [Veröffentlichung](/docs/data_publication) im experimentellen Teil einer Dissertation oder eines Artikels eignet. <br/> Außerdem sind die Daten maschinenlesbar und damit leicht auffindbar, was ein Aspekt für [FAIR data](/docs/fair) ist (**F**indable, **A**ccessible, **I**nteroperable, **R**eusable). Auch die weiteren Aspekte sollten erfüllt sein, dies erhöht die maschinelle Verwertbarkeit für die zukünftige Nutzung der Daten. Ein weiterer Vorteil eines [ELN](/docs/eln) ist, dass es mit einem [Repositorium](/docs/repositories) gekoppelt werden kann, was den [Publikationsprozess](/docs/data_publication) erleichtern wird. [Wählen Sie ein Repositorium](/docs/choose_repository), das für Ihre Daten geeignet ist.
:::


## Datenverarbeitung

:::danger Warum:          
Die gewonnenen Rohdaten müssen für die Interpretation aufbereitet werden, z.B. durch Integration von NMR-Peaks oder Auslesen von IR-Banden.
:::

:::info Wie:          
Die Daten können mit unterschiedlicher Software für jede Methode verarbeitet werden. Aber ein [ELN](/docs/eln) kann Ihnen ein Software-Tool für die Verarbeitung verschiedener Methoden zur Verfügung stellen. In [Chemotion](https://www.chemotion.net/chemotionsaurus/index.html) ist das Softwaretool Chemspectra integriert, das zur Zeit für NMR und IR verwendet werden kann. Weitere Methoden werden in Zukunft ergänzt.
:::

:::caution Considerations:         
Durch die [Dokumentation](/docs/data_documentation) und die Bearbeitung der Daten im selben Softwaretool werden Synergien erzielt. Zum Beispiel kann Chemspectra automatisch kontrollieren, ob das Integral des NMR-Spektrums zum Zielmolekül der angeschlossenen Synthese passt. Nach der Bearbeitung Ihrer Daten in Chemspectra können Sie Ihre Daten in einer standardisierten Peakliste exportieren, die in den experimentellen Teil einer Dissertation oder einer [Publikation](/docs/data_publication) eingefügt werden kann.
:::


## Gemeinsame Nutzung, Bewahrung und Wiederverwendung von Daten

:::danger Warum:      
Die Zusammenarbeit ist einfacher, wenn [gut organisierte Daten](/docs/data_organisation) zugänglich sind. Für weitere Forschungen können frühere Forschungen überprüft werden und bereits vorhandene Daten sind wiederverwendbar.
:::

:::info Wie:          
Ein [ELN](/docs/eln) stellt eine leistungsfähige interne Datenbank dar. Innerhalb von [Chemotion](https://www.chemotion.net/chemotionsaurus/index.html) können die eigenen Ergebnisse nach Struktur oder Stichwort durchsucht werden. Außerdem können wissenschaftliche Suchmaschinen wie SciFinder in eine [ELN](/docs/eln) implementiert werden. 
:::

:::note Hinweis:        
Durch die Nutzung des [Chemotion ELN](https://www.chemotion.net/chemotionsaurus/index.html) werden Forschungsdaten und zugehörige [Metadaten](/docs/metadata) direkt in das [Chemotion repository](https://www.chemotion-repository.net/welcome) übertragen und automatisch mit anderen [Repositorien](https://www.re3data.org/) geteilt.
:::
